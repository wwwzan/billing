## 三、关键逻辑思路

### 3.1 阶梯电价计算

采用分段累加的方式计算跨阶梯电费，各阶梯单价如下：

```
第一阶梯（0-200度）：     单价 0.50元/度
第二阶梯（200-400度）：   超过部分单价 0.80元/度
第三阶梯（400度以上）：   超过部分单价 1.20元/度
```

计算逻辑：
```
用电量 ≤ 200度：   电费 = 用电量 × 0.50
200度 < 用电量 ≤ 400度： 电费 = 200×0.50 + (用电量-200)×0.80
用电量 > 400度：   电费 = 200×0.50 + 200×0.80 + (用电量-400)×1.20
```

### 3.2 峰谷时段判断与费率调节

时段定义：
```
高峰时段：(8:00-22:00]  - 总费率增加10%（×1.10）
低谷时段：(22:00-次日8:00] - 总费率减少20%（×0.80）
```

关键区别：与简单的"电价倍率"不同，本实现采用"总费率调节"方式，即在基础阶梯电费计算完成后，对总电费进行百分比调整。

### 3.3 时间输入解析

支持多种时间输入格式：
- 标准格式：`"14:00"`、`"08:30"`
- 简写格式：`"14"`、`"8"`
- 自动处理前后空格

## 六、跨阶梯计费逻辑难点

### 6.1 难点描述

在实现跨阶梯计费时，最大的难点在于**如何正确处理用电量跨越多个阶梯的情况**，特别是要准确理解"超过部分单价"的含义。例如，当用电量为500度时，需要：

1. 正确识别用电量跨越了三个阶梯
2. 对每个阶梯内的用电量**分别**应用对应的单价
3. 确保阶梯边界的计算不重复、不遗漏
4. 理解"超过部分单价"意味着只有超出该阶梯阈值的部分才使用更高单价

### 6.2 解决方案

采用**分段累加**的策略，从低到高依次处理每个阶梯：

```go
func CalculateTieredCharge(usage float64) float64 {
    var charge float64 = 0.0

    if usage <= 0 {
        return 0.0
    }

    // 第一阶梯：0-200度，全部按0.50元计算
    if usage <= Threshold1 {  // 200度
        return usage * Price1  // 全部用电量 × 0.50
    }
    charge = Threshold1 * Price1  // 第一阶梯满额：200 × 0.50

    // 第二阶梯：200-400度，超过200的部分按0.80元计算
    if usage <= Threshold2 {  // 400度
        charge += (usage - Threshold1) * Price2  // 超过部分 × 0.80
        return charge
    }
    charge += (Threshold2 - Threshold1) * Price2  // 第二阶梯满额：200 × 0.80

    // 第三阶梯：400度以上，超过400的部分按1.20元计算
    charge += (usage - Threshold2) * Price3  // 超过部分 × 1.20

    return charge
}
```

### 6.3 关键点总结

1. **阶梯边界理解**：200度和400度是阈值，不是"超过即全部按高价计算"
   - 错误理解：201度全部按0.80算 = 160.8元
   - 正确理解：200度按0.50 + 1度按0.80 = 100.8元

2. **分段累加原则**：每个阶梯只计算该阶梯范围内的电量

3. **边界精确性**：确保200度、400度等边界值的计算精确无误
   - 200度：200 × 0.50 = 100元
   - 400度：200 × 0.50 + 200 × 0.80 = 260元

4. **代码结构**：采用"先判断、后累加"的方式，避免复杂的嵌套条件

- 14:00属于高峰时段，总费率+10%
- 最终电费：260 × 1.10 = 286.00元
