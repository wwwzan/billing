```go
const (
    Threshold1 = 200.0  // 第一阶梯上限

    Price1 = 0.50  // 第一阶梯：0-200度，0.5元/度
```

这里要注意，第二档和第三档的单价是针对"超过部分"的，不是全部用电量都按这个价算。

接下来是峰谷时段的调节因子：


这里用的是"调节因子"而不是"倍率"，意思是在基础电费算完之后，对总费用进行百分比调整。

### 第三部分：时段判断

```go
func GetTimePeriod(hour int) TimePeriod {
```

注意这里的区间是左闭右开的，8点算高峰，22点算低谷。

### 第四部分：核心计费逻辑

这是整个程序最核心的函数，CalculateTieredCharge：

```go
func CalculateTieredCharge(usage float64) float64 {
    var charge float64 = 0.0

    if usage <= 0 {
        return 0.0
    }

    // 第一阶梯：0-200度
    if usage <= Threshold1 {
        return usage * Price1
    }
    charge = Threshold1 * Price1  // 第一阶梯满额：100元

    // 第二阶梯：200-400度
    if usage <= Threshold2 {
        charge += (usage - Threshold1) * Price2
        return charge
    }
    charge += (Threshold2 - Threshold1) * Price2  // 第二阶梯满额：160元

    // 第三阶梯：400度以上
    charge += (usage - Threshold2) * Price3

    return charge
}
```

这个函数采用分段累加的方式。举个例子，如果用电量是500度：
- 先算第一阶梯的满额：200度 × 0.5 = 100元
- 再算第二阶梯的满额：200度 × 0.8 = 160元
- 最后算第三阶梯：100度 × 1.2 = 120元
- 总计：380元

关键是每一档只算自己这一档的电量，不是全部按最高价算。

## 跨阶梯计费的难点（30秒）

最后想说一下跨阶梯计费的难点。最容易出错的地方是理解"超过部分单价"这个概念。

比如201度电，错误的理解是：201度全部按0.8元算，得160.8元。

正确的理解是：200度按0.5元算，1度按0.8元算，得100.8元。

代码中用分段累加的方式，就是为了避免这种错误。每一档只算自己这一档的电量，清晰明了。
